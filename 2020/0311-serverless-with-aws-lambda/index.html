- What are all these buzzwords? (FaaS / Serverless) - Ask if someone can define serverless - We're moving towards more granular deployments of code - Explain process

"Serverless"


  • Upload your code to Platform
  • Platform coordinates execution of your code
    • SNS (Simple Notification Service)
    • SQS (Simple Queue Service)
    • CloudWatch Event
    • API Gateway
    • Manual trigger

- Seems to be cost effective for low volume apps

Okay, but why?

- No servers (nothing to maintain or keep running) - Pay per execution (many platforms offer freemium pricing) - Automatic scaling - Edge deployment of your code (limited) - [AWS] Easy access to DynamoDB - [AWS] Supports Java, Go, PowerShell, Node.js, C#, Python, and Ruby - Websockets?

- Need to focus on stateless - Unsure how costs will actually scale

Then why not?

- No servers (start-up time is critical) - Pay per duration-execution (memory consumption and cpu time matter) - Vendor Lock-in - [AWS] Dealing with binary dependencies is annoying - Debugging can be difficult - Time limit on calls (900s) - [AWS] Lose the ability to run on the edge if you need access to your VPC
